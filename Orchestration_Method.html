The Orchestration Method

Orchestrate, don't carry the load. A single person directs a manager agent that conducts swarms of specialized sub-agents — to plan and build work too large for any one mind to hold — and converges on truth from many independent angles.

The shift: from one agent grinding a task forward and leaving gaps — to one agent conducting many and converging on what's true.

The foundation

Two laws hold the whole method up.

Everything that follows — the flowchart, the agent types, the audits — is machinery for obeying these two laws. Read nothing else and you still have the method.

LAW 1

Orchestrate, don't carry the load.

The manager agent spends its limited attention orchestrating, aggregating, digesting, and monitoring — never delivering the work directly. It composes briefings, fires sub-agents, folds their results, and watches for drift. The actual studying, deriving, and building happens in fresh sub-agent contexts, so the manager never fills up its own mind with raw work content.

"What cannot be held in mind cannot be reasoned over." Context is working capital — spend it conducting, not consuming.

LAW 2

Plan to convergence before you execute.

You never jump straight to building. You plan until many independent angles agree — multi-dimensional convergence — and only then do you execute. And here is the part people miss: the same orchestrator mode governs both phases. The execute phase runs the identical wave method as the plan phase — typed sub-agents, audits, convergence checks — not a sudden switch into "just code it now."

Plan and build are two stretches of one continuous discipline, not two different modes.

The brake

Two layers: a mindset that's always on, machinery that fires on cue.

The method has a built-in brake so it never over-engineers a simple request. The disposition runs everywhere; the heavy machinery only fires for genuinely deep work.

LAYER 1 · always on

The orchestrator mindset

Address every task as an orchestrator, not a worker. Use search-and-sample to find what matters; read a targeted slice, not a whole file, when a slice will do. Aggregate and digest rather than consume raw content. This needs no sub-agents at all — it is just how careful work is done, even on the smallest task.

LAYER 2 · triggered

The wave machinery

The full chain — study → audit → plan → audit-the-plan → digest → audit-the-digest → execute → test → audit → audit-the-audit → adversarial → independent secondary audit — fires only on an explicit trigger or on confirmed deep-scope work (and then only after surfacing the wave plan for a go/no-go).

The boundary — depth scales to scope

Simple tasks stay simple. You do not spawn a swarm to answer "what does this function mean?" or to make a single one-line edit. A single read, a single reply, a small bounded fix — those run directly with the Layer-1 mindset and nothing more. The machinery is reserved for campaigns, plans, investigations, and multi-surface changes. When in doubt, surface a two-line wave plan and ask before firing.

The cast

The agent-type taxonomy — who does what, and the rules that keep them honest.

A swarm is only as good as its role discipline. Each sub-agent is typed at dispatch: it gets a narrow job, an isolation rule that keeps it from contaminating other roles, and a model tier. (Throughout, a seat just means one such typed sub-agent instance doing one job.) The isolation rules are the load-bearing part — they are what make the audits independent and the math trustworthy.

The unit of autonomy

The dispatch — a complete briefing a stranger could run cold.

Autonomy lives here. Every sub-agent starts with no memory of the conversation. The reason the loop can run for a long stretch without the operator stepping in between stages is that each dispatch is self-contained: mission, verbatim invariants, the prior stage's output, hard rules, what not to touch, and the exact shape of the answer. A fresh agent needs nothing else to do the job correctly.

# ── DISPATCH · STAGE 3 · PLAN ── §0 MISSION FRAMING why this seat exists §1 TASK INVARIANT echoed VERBATIM — "do not widen a guard; fix the input" …overrides any template default §2 PRIOR-STAGE OUTPUT the studied substrate, pinned to exact file:line citations §3 HARD RULES the guardrails §4 FILES NOT TO TOUCH explicit do-not list §5 REQUIRED RETURN SHAPE exact filename + numbered sections, so the next stage consumes it cleanly §6 SEAT TYPE + MODEL TIER
  • Mission framing. One paragraph on why this seat exists and what "done" looks like.
  • The task invariant, verbatim. The non-negotiable rules are quoted word-for-word and override every template default — so intent can't drift through paraphrase.
  • The prior stage's output. The studied substrate handed forward, pinned to exact file:line citations — the agent never picks its own inputs.
  • Hard rules + a "files not to touch" list. The guardrails that keep a cold agent inside its lane.
  • The required return shape. An exact filename and numbered sections, so the next stage can consume the result mechanically — no human re-typing.
  • Typed seat + model tier. Which role it plays and on which model.
Why this buys autonomy

Because every dispatch is a full cold-start briefing and stages close internally on convergence (an auditor's accept, not the operator's sign-off), the manager can fire a stage and step away. The operator sets the pace — which work fires and how much at once — but does not drive each individual step.

The core idea

Multi-dimensional convergence — many angles, one answer.

This is the engine's heart. For each question, independent voices investigate separately and never read each other — like asking witnesses to describe a scene in isolation. A fresh synthesis seat then measures how much they agree. Agreement, not authority, is what greenlights the next stage.

The convergence meter

A synthesis seat (never one of the voices) tags each finding and issues a verdict (the bands below are illustrative, not a fixed contract):

≥ 80%
CONVERGE
The voices land together — promote with confidence.
50–80%
PARTIAL
Real-but-missed items exist — a closer seat re-checks against the actual source.
< 50%
DIVERGE
The question itself is suspect — escalate it as a design problem.
The human's scarce contribution is doubt

An AI converges confidently but shallowly — and it will not doubt its own premise. It answers the question as asked. The single most valuable thing the human brings is doubt: the willingness to ask whether the premise was even right. So the rule is firm — never "lock and move on" on first agreement within a single line of audit. First convergence is the start of scrutiny, not the end of it.

An unresolvable question is a symptom, not a matter of taste

When an item won't resolve — when the agents reach a forced pick with no evidenced winner — that is treated as a symptom of a design mistake, not as a judgment call to bounce up to the operator. The response is to re-investigate the design until the question dissolves, rather than make the human guess. Only what is truly irreducible ever reaches the operator, and it arrives as an evidenced pick-sheet on grounded footing.

Defense in depth

The audit superstructure — and the standing watch.

A single audit has a single blind spot. So high-stakes work is checked from several deliberately different angles, and a trio of auditors rides along every wave watching for erosion.

The layered audit chain

  • Primary independent audit — framing A; verifies the deliverable against the source. The author never audits their own work.
  • Adversarial audit — framing B; hunts what the primary would not catch: cherry-picking, hidden assumptions, framing drift.
  • Audit-of-audit — confirms the audit itself was complete and well-scoped.
  • Independent secondary — mandatory after planning or code; fresh context, a different framing — sometimes a different model or engine entirely.
  • Roving checks — time-travel, plan-break, and rules/memory probes, fired alongside the waves.

The standing watch — re-run every wave

Three auditors are re-invoked on every single wave, doing nothing but protecting the truth from erosion. Folds are additive — they add without dropping.

REFERENCE-ANCHOR

Checks each wave's output against the frozen reference set of locked premises.

VOCABULARY-DRIFT

Catches a new name invented for an existing concept before it splits the work.

NOTHING-LOST

Diffs against the previous version so no load-bearing item is silently dropped.

The executable-prototype gate

Before any real change, a throwaway prototype is built against a synthetic case. Its job is not to work — its job is to force hidden specification ambiguities to surface mechanically. Every "wait, which behavior did we mean here?" the prototype exposes is value extracted. Finding zero ambiguities is itself a red flag — it usually means the prototype wasn't probing hard enough, not that the spec was perfect.

Sidebar — the seven-round sliced audit

For the highest-stakes deliverables, a single audit becomes a seven-round pattern:

  1. Worker produces the deliverable.
  2. Layer-sliced auditors each verify one horizontal layer.
  3. Axis-sliced advisors each review one vertical concern.
  4. Aggregation folds them — and never collapses dissent into false agreement.
  5. Audit-the-audit checks the aggregation was complete.
  6. Additive fold merges accepted findings without dropping anything.
  7. Delivery-verify confirms the final artifact matches the contract.

Layer 2.5 · the ultimate form

Paired orchestration — cooperation under one shared ledger.

For the largest builds, two peer managers run at once — and the names carry the contract. The Executor runs the campaign and is the only instance that edits the tree or commits. The Shadow is not a sub-agent — it is a peer, running its own thread and its own waves, that reads the source fresh and challenges whether the Executor's premises are actually true, while the work is still steerable — and touches nothing but the ledger. The Shadow can even be a different model or engine, for a genuinely independent framing. They speak only through one append-only, author-tagged Coordination Ledger, and coordination flows both ways — this is cooperation, not surveillance.

OBSERVATION · no ack needed STEER · advisory, must respond BLOCKING · halts that unit ACK · acted on it DEFER · parked with reason ESCALATE · to the operator

The contract is advisory + mandatory-ack: a mark is advisory, but no peer may silently ignore one. Every mark must be met with an ACK-and-act, a DEFER-with-reason, or an ESCALATE. Unresolved BLOCKING marks escalate straight to the operator. An illustrative exchange:

#L-014 OBSERVATION [SHADOW]
Re-walked the spec from source. The normalization divisor in §4 is described two different ways across the plan and the design doc — flagging for awareness before it hardens.
#L-015 STEER [SHADOW]
TIME-CRITICAL. The dispatch pins a fixed divisor, but the architecture says full-scale should be measured empirically per stratum. A fixed value sits below the real scale — the factor would pin to its neutral value and silently re-create the exact flaw this work exists to remove. Don't lock the fixed divisor.
#L-016 ACK [EXEC]
Your catch converges with my own seat's independent verdict — triangulation across two voices. RESOLUTION ADOPTED: empirical per-stratum scale. The fixed divisor is dead.
#L-021 DEFER [EXEC]
Logging-format nit acknowledged but parked — out of scope for this unit; filed for the cleanup pass with a tracking note. Reason recorded so it isn't lost.
#L-027 BLOCKING [SHADOW]
Two source modules disagree on the contract this unit depends on. Cannot verify either reading is safe. Halting this unit — needs an operator call on which contract is canonical.
#L-028 ESCALATE [EXEC]
Confirmed — the disagreement is real and I can't resolve it from source. Escalating #L-027 to the operator with both readings laid out and the safer default recommended.
THE 3-AGENT FAMILY · the Senior backstop

When context thins, the role outlives the instance

A manager's context window is finite. When it runs low, it hands its active load to a fresh same-role successor — cold-started from a handover doc plus the ledger — and the outgoing instance becomes the Senior: a light-touch backstop that carries the full history at near-zero ongoing cost. The Senior defers to the active pair and steps in only on a trigger — an escalation, a peer disagreement, an active agent gone quiet, or an unaudited commit — on a periodic (~20-min) cadence. Roles persist across instances, so the run becomes a three-agent family: Executor + active Shadow + Senior.

DYNAMIC LOAD-SHARING · still exploring

The Shadow can carry real load — safely

The Shadow need not only audit. The collision-safe default splits work by phase: the Shadow owns scope · settle · verify (next-chapter file-disjointness maps, math-seam settling, spec drafting, deep verification, decision-package prep); the Executor owns build · commit. If the Shadow also edits the tree — true co-build — that relaxes the one-editor rule, and is allowed only with all four guards: a declared disjoint file-partition, a single committer, operator confirmation, and an independent verifier of anything the Shadow authored.

The Senior speaks in the same ledger, in the same marks — here, a periodic sweep catches a commit that slipped through unaudited:

#L-033 STEER [SENIOR]
Periodic sweep. The commit at #L-031 landed with no audit mark against it, and the Shadow has been quiet for two cycles. Hold further commits until the Shadow confirms it verified #L-031 — or the operator clears it.
#L-034 ACK [EXEC]
Acknowledged — holding the line. Shadow, please confirm #L-031 against source.

Why it holds

The load-bearing invariants.

Cooperation between agents is only safe because a few rules never bend. These are what let two or three peer instances move fast on one codebase without clobbering each other — or shipping a bug a single confident pass called “sound.”

ONE EDITOR · NON-COLLISION

Exactly one instance edits the tree and commits. That single rule buys clobber-safety and a clean, linear commit trail — you always know which voice authored what.

THE SECOND FRAMING IS THE VALUE

An independent framing catches what one framing — or the author — cannot. In one real run a cross-file key-mismatch sailed past the per-file read, but the holistic secondary audit, corroborated by the shadow, caught it before it shipped.

BUILD BEHIND GATES

Work is built uncommitted and reversible, gated on a secondary audit plus the operator's sign-off. Nothing is saved until two independent sets of eyes and the operator agree — so a missed bug never ships.

FILE-DISJOINT PARALLEL · ONE TREE

Parallel work units are partitioned by file in the one main tree, each committed independently — so any held or gated unit stays reversible. No worktrees, no invented isolation.

OPERATOR-RESERVED DECISIONS

Choices that belong to the operator surface with evidence, at their point of evidence. Agents prepare and float them — they never self-sign the operator's doubt-gates.

The discipline that matters most

When are you allowed to build?

The single most important judgment in the whole method: a plan is not build-ready because it is written. It is build-ready because it has survived independent attempts to break it. You build only after the converged plan clears the full gauntlet.

Real autonomy vs. the illusion of progress

A single confident forward-pass covers distance — it produces output fast and feels like progress. But it leaves gaps and potholes nobody graded. Real autonomy is the opposite: slower to start, because the plan must survive being attacked from many sides first — but what ships has been seen from every angle that matters. The cost of the gauntlet is paid once, up front; the cost of a silent gap is paid forever, downstream.

A single case study

One real catch — why the method earns its cost.

SHADOW · PEER PREMISE CHECKED AGAINST SOURCE CROSS-MODEL CORROBORATION

The buried default that two independent voices caught.

In a real planning campaign, a main orchestrator run had marked a flagship decision "settled — no operator choice needed." A second, peer shadow manager — running live alongside it and checking the claim against the actual source code — found that signing off would have silently locked in a choice the operator never knowingly made: a quietly-dropped fallback meant one specific formula would become permanent by default. A single confident forward-pass ships exactly that, invisibly. What caught it was structure: an independent voice checking premises against source while the work was still steerable.

Then a third voice — a different model entirely — reviewing the same question a day later, landed on the very same hidden flaw. Two independent voices converging on one buried defect is a conviction no single pass can manufacture. That is the moment the method earns its cost.

In one breath: One person runs deep work as a swarm of AI agents instead of doing it alone. The manager agent never carries the work in its own head — it orchestrates: it splits the problem into pieces, sends typed sub-agents (scouts, workers, mathematicians, auditors) to study and build each piece in isolation, then folds their results and measures how much independent angles agree. Agreement — not the manager's say-so — is what lets the work advance.

Crucially, the same mode governs planning and building. You don't drop out of orchestration to "just code it"; the execution phase runs the identical wave method as the planning phase, with the same audits and convergence checks. Every deliverable is verified by a different agent, attacked adversarially, re-checked by a differently-framed secondary (sometimes a different model or engine), and — for the largest builds — watched live by a peer shadow manager through a shared, append-only ledger where the main manager must answer every challenge rather than silently ignore it.

The discipline scales to the work: a one-line question stays a one-line answer; a campaign earns the full machinery and a paired run. And the gate is firm — you build only after the plan has survived independent attempts to break it, because a confident forward-pass merely covers distance while leaving silent gaps, whereas convergence across many angles is a conviction no single pass can manufacture. That is the method.
